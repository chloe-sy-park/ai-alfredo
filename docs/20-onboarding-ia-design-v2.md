## 🚀 온보딩 플로우 설계

### 전체 구조

```
[1. 웰컴] → [2. 가치 소개] → [3. 방향 선택] → [4. 개입 방식] → [5. 성장형] → [홈]
   │            │              │              │              │
 관계 형성    기능 이해      개인화 시작     기대치 설정     성장 약속
```

### Step 1: 웰컴 (관계 형성)

**목적**: AlFredo가 단순 도구가 아닌 "관계"임을 인식시킴

```
┌─────────────────────────────────┐
│                                 │
│           🐧                    │
│                                 │
│        AlFredo                  │
│                                 │
│   "도구가 아니라 관계입니다"      │
│                                 │
│   당신을 키우는 AI 멘토이자,      │
│   당신이 키우는 퍼스트 펭귄       │
│                                 │
│                                 │
│   [      시작하기      ]        │
│                                 │
│         건너뛰기                 │
│                                 │
└─────────────────────────────────┘
```

**컨텐츠**:
- 헤드라인: "AlFredo"
- 서브타이틀: "도구가 아니라 관계입니다"
- 설명: "당신을 키우는 AI 멘토이자, 당신이 키우는 퍼스트 펭귄"
- CTA: "시작하기" (검정 또는 노란 버튼)
- Skip: "건너뛰기" → 홈으로

### Step 2: 가치 소개 (기능 이해)

**목적**: AlFredo가 제공하는 3가지 핵심 가치를 명확히 전달

```
┌─────────────────────────────────┐
│                                 │
│  AlFredo는 이렇게 도와줍니다     │
│                                 │
│  ┌───────────────────────────┐  │
│  │ 📋 오늘 할 일 우선순위      │  │
│  │                           │  │
│  │ 마케팅 보고서 검토 → 팀 미팅 │  │
│  │ → 디자인 문서 작성          │  │
│  │                           │  │
│  │ 이후엔 일석삼조 앞뒤로 배치  │  │
│  └───────────────────────────┘  │
│                                 │
│  ┌───────────────────────────┐  │
│  │ 🎯 지금 집중할 것           │  │
│  │                           │  │
│  │ 마케팅 보고서 최종 검토     │  │
│  │ 45분 소요 예상             │  │
│  │                           │  │
│  │ 당일의 작업 목록을 분석해서  │  │
│  └───────────────────────────┘  │
│                                 │
│  ┌───────────────────────────┐  │
│  │ 📊 이번 주 패턴             │  │
│  │                           │  │
│  │ 오후 3-5시 사이 집중력 하락  │  │
│  │ 짧은 휴식 필요              │  │
│  │                           │  │
│  │ 매일 리듬 측정              │  │
│  └───────────────────────────┘  │
│                                 │
│   [        다음        ]        │
│                                 │
└─────────────────────────────────┘
```

**3가지 핵심 가치**:
1. **우선순위 정리** - "오늘 할 일 우선순위"
2. **집중 관리** - "지금 집중할 것"
3. **패턴 분석** - "이번 주 패턴"

### Step 3: 방향 선택 (개인화 시작)

**목적**: 사용자 맥락 파악 + 개인화 기반 마련

```
┌─────────────────────────────────┐
│                                 │
│  어떤 방향으로 시작할까요?        │
│                                 │
│  ┌───────────────────────────┐  │
│  │ 🏢 일 중심                  │  │
│  │    업무 생산성에 집중하고 싶어요│  │
│  └───────────────────────────┘  │
│                                 │
│  ┌───────────────────────────┐  │
│  │ ❤️ 삶 중심                  │  │
│  │    생활 패턴과 밸런스가 중요해요│  │
│  └───────────────────────────┘  │
│                                 │
│  ┌───────────────────────────┐  │
│  │ ⚖️ 둘 다                    │  │
│  │    일과 삶 모두 챙기고 싶어요 │  │
│  └───────────────────────────┘  │
│                                 │
│  ┌───────────────────────────┐  │
│  │ 🐧 잘 모르겠어요             │  │
│  │    AlFredo가 판단해주세요    │  │
│  └───────────────────────────┘  │
│                                 │
└─────────────────────────────────┘
```

**선택지 → 초기 설정 영향**:

| 선택 | 홈 기본 모드 | 브리핑 톤 | 우선순위 가중치 |
|------|------------|----------|---------------|
| 일 중심 | Work Mode | 생산성 중심 | 업무 태스크 ↑ |
| 삶 중심 | Life Mode | 웰빙 중심 | 라이프 이벤트 ↑ |
| 둘 다 | Balanced | 균형 잡힌 | 동등 |
| 잘 모르겠어요 | Auto | 적응형 | AI 판단 |

### Step 4: 개입 방식 (기대치 설정)

**목적**: AlFredo가 어떻게 "개입"하는지 미리 알려줌 → 거부감 감소

```
┌─────────────────────────────────┐
│                                 │
│  AlFredo는 이렇게 개입합니다     │
│                                 │
│  ┌───────────────────────────┐  │

│  │ 중요한 작업 계속 미루고 있을 때│  │
│  │                           │  │
│  │ 🐧 AlFredo                 │  │
│  │ "이 작업은 오늘 3시까지     │  │
│  │  1개 완료 필요.            │  │
│  │  지금 시작하시겠어요?"      │  │
│  └───────────────────────────┘  │
│                                 │
│  ┌───────────────────────────┐  │
│  │ 상황                       │  │
│  │ 연속으로 3시간 일했을 때     │  │
│  │                           │  │
│  │ 🐧 AlFredo                 │  │
│  │ "2시간 이상 집중했습니다.   │  │
│  │  10분 휴식을 추천합니다."   │  │
│  └───────────────────────────┘  │
│                                 │
│  ┌───────────────────────────┐  │
│  │ 상황                       │  │
│  │ 저녁 시간에 업무 요청이 들어올 때│  │
│  │                           │  │
│  │ 🐧 AlFredo                 │  │
│  │ "현재 Life 모드입니다.      │  │
│  │  내일 오전에 처리하면 어떨까요?"│  │
│  └───────────────────────────┘  │
│                                 │
│   [      이해했어요      ]      │
│                                 │
```

**3가지 개입 시나리오**:
1. **미루기 방지** - 데드라인 리마인더
2. **번아웃 방지** - 휴식 권유
3. **경계 보호** - 워라밸 가드

### Step 5: 성장형 (성장 약속)

**목적**: 장기적 관계 형성 + "함께 성장" 기대감

```
┌─────────────────────────────────┐
│                                 │
│        AlFredo는 성장합니다      │
│                                 │
│         ┌──────────────┐        │
│         │              │        │
│         │   🐧  5%     │        │
│         │   ████░░░░░░ │        │
│         │              │        │
│         └──────────────┘        │
│                                 │
│   처음에는 5% 정도만 알고 있습니다.│
│   하지만 함께하는 시간만큼        │
│   정확해집니다.                  │
│                                 │
│   ─────────────────────────     │
│                                 │
│   AlFredo를 키우는 과정에서,     │
│   AlFredo는 당신도 성장시킵니다.  │
│                                 │
│                                 │
│   [     함께 시작하기     ]      │
│                                 │
└─────────────────────────────────┘
```

**핵심 메시지**:
- "처음엔 5%만 알고 있습니다" → 완벽하지 않음을 인정
- "함께하는 시간만큼 정확해집니다" → 학습하는 AI
- "당신도 성장시킵니다" → 상호 성장 관계

## 🔄 네비게이션 재설계

### 기존 vs 제안

| 구분 | 기존 | 제안 |
|------|------|------|
| **메인 네비** | 하단 탭 5개 (항상 노출) | 햄버거 → 오른쪽 드로어 (숨김) |
| **플로팅** | FAB 펭귄 1개 | 채팅 입력 + 퀵액션 (통합 바) |
| **핵심 인터랙션** | 탭 전환 | 채팅 + 퀵액션 |
| **화면 활용** | 하단 60px 점유 | 전체 화면 활용 |

### 드로어 인터랙션

```
열기:
├── 햄버거 버튼 탭 → 오른쪽에서 슬라이드 인 (300ms)
├── 화면 오른쪽 edge 스와이프 → 드래그로 열기
└── 페이지 어디서든 가능

닫기:
├── 오버레이 영역 탭 → 슬라이드 아웃
├── 메뉴 아이템 선택 → 자동 닫힘 + 페이지 전환
├── 왼쪽으로 스와이프 → 드래그로 닫기
└── 뒤로가기 제스처/버튼
```

### 레벨별 상세 정의

#### Level 0: 앱 진입점

| 화면 | 조건 | 다음 단계 |
|------|------|----------|
| 스플래시 | 항상 | 0.5초 후 자동 전환 |
| 온보딩 | 첫 실행 | 5단계 완료 후 홈 |
| 홈 | 재방문 | 바로 홈 |

#### Level 1: 홈 (Central Hub)

```
홈의 역할:
├── 정보 허브: 모든 핵심 정보 집약
├── 의사결정 지원: "지금 뭘 해야 하지?"에 답변
├── 네비게이션 시작점: 드로어/플로팅으로 확장
└── 상태 모니터링: 실시간 브리핑
```

**홈에서 가능한 인터랙션**:

| 인터랙션 | 결과 | 레벨 이동 |
|----------|------|----------|
| 브리핑 더보기 | 상세 브리핑 바텀시트 | Level 1.5 |
| Right Now 시작 | 타이머 모달 | Level 1.5 |
| 모드 카드 클릭 | 해당 OS로 이동 | Level 2 |
| 타임라인 아이템 클릭 | 상세 바텀시트 | Level 1.5 |
| 햄버거 메뉴 | 드로어 오픈 | Overlay |
| 채팅 입력 | 채팅 시작 | Overlay |
| 퀵액션 | 메뉴 확장 | Overlay |

#### Level 1.5: 오버레이 (모달/바텀시트)

```
바텀시트 유형:
├── 상세 보기 (반 높이)
│   └── 브리핑 상세, 일정 상세, 태스크 상세
├── 입력 폼 (전체 높이)
│   └── 태스크 추가, 메모 추가, 일정 추가
└── 선택 (작은 높이)
    └── 컨디션 선택, 타이머 설정
```

#### Level 2: OS/섹션 페이지

| 페이지 | 접근 방법 | 핵심 기능 |
|--------|----------|----------|
| 캘린더 | 드로어 메뉴 | 월/주/일 뷰, 일정 관리 |
| 워크OS | 드로어 or 모드카드 | 업무 태스크, 시그널 관리 |
| 라이프OS | 드로어 or 모드카드 | 웰빙 상태, 습관 트래킹 |
| 리포트 | 드로어 메뉴 | 주간 리뷰, 패턴 분석 |

#### Level 3: 상세 페이지

```
상세 페이지 유형:
├── 바텀시트 상세 (Level 2에서 열림)
│   └── 태스크 수정, 일정 수정
├── 풀페이지 상세 (복잡한 정보)
│   └── 리포트 상세 분석, 패턴 드릴다운
└── 외부 연결
    └── 구글 캘린더, 이메일 앱
```

## 🔄 인터랙션 패턴

### 페이지 전환 패턴

```
홈 ↔ 다른 페이지:
├── 드로어에서 선택 → 페이드 전환 (200ms)
├── 모드 카드 탭 → 슬라이드 업 전환 (300ms)
└── 뒤로가기 → 슬라이드 다운 전환

바텀시트:
├── 열기 → 아래에서 슬라이드 업
├── 닫기 → 아래로 드래그 or 바깥 탭
└── 높이 조절 → 드래그로 반높이 ↔ 전체 높이
```

## 🎯 설계 원칙

| 원칙 | 설명 |
|------|------|
| **홈 = 허브** | 모든 핵심 정보를 홈에서 한눈에 |
| **1-tap 접근** | 주요 기능은 1번 탭으로 도달 |
| **맥락 우선** | 시간/상황에 따른 정보 자동 조정 |
| **최소 인지부하** | ADHD 친화적 단순 UI |
| **펭귄 = 도우미** | FAB로 언제든 대화 가능 |
| **채팅 중심** | 하단 탭 제거, 채팅 바가 핵심 인터랙션 |
| **숨김 네비** | 드로어로 화면 활용 극대화 |
